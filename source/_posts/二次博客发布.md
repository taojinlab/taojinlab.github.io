# 二次博客发布

<!-- more -->

上一篇记录了 Hexo 博客从零创建到发布的完整流程。以后再次发布新文章时，不需要重新搭建博客，只需要按照固定步骤操作即可。

## 第一步 进入博客目录

打开 PowerShell，进入本地博客文件夹：

```powershell
cd "D:\code data\my-blog"
```

## 第二步 新建文章

使用 Hexo 新建一篇文章：

```powershell
npx hexo new "文章标题"
```

例如：

```powershell
npx hexo new "我的第二篇博客"
```

新文章会生成在：

```text
D:\code data\my-blog\source\_posts
```

进入这个文件夹，找到刚刚生成的 Markdown 文件，然后编辑文章内容。

## 第三步 本地预览

文章写好后，先在本地预览：

```powershell
npx hexo server
```

浏览器打开：

```text
http://localhost:4000/
```

检查文章标题、正文、排版、链接是否正常。

预览结束后，在 PowerShell 中按：

```text
Ctrl + C
```

停止本地预览服务。

## 第四步 保存源码到 GitHub

确认文章没有问题后，先把 Markdown 原文和博客源码保存到 GitHub 的 `main` 分支：

```powershell
git add .
git commit -m "Add new post"
git push
```

这一组命令保存的是博客源码，包括：

```text
Markdown 文章
Hexo 配置
主题配置
package.json
```

源码保存在 GitHub 的 `main` 分支，方便以后换电脑、恢复博客或查看修改记录。

## 第五步 生成网页

执行：

```powershell
npx hexo clean
npx hexo generate
```

其中：

```text
npx hexo clean：清理旧的生成文件和缓存
npx hexo generate：把 Markdown 文章生成静态网页
```

生成后的网页文件会放在：

```text
D:\code data\my-blog\public
```

`public` 文件夹里的内容才是真正给浏览器访问的网页文件，例如：

```text
index.html
archives
tags
css
js
```

## 第六步 发布网站

执行：

```powershell
npx hexo deploy
```

这一步会把 `public` 文件夹里的网页文件发布到 GitHub 仓库的 `gh-pages` 分支。

本博客的发布配置在 `_config.yml` 中：

```yaml
deploy:
  type: git
  repo: https://github.com/taojinlab/my-blog.git
  branch: gh-pages
```

这段配置的意思是：

```text
使用 git 方式发布
发布到 taojinlab/my-blog 仓库
发布目标分支是 gh-pages
```

所以执行 `npx hexo deploy` 时，Hexo 会通过 `hexo-deployer-git` 插件，把生成好的网页上传到 `gh-pages` 分支。

## 第七步 访问博客

发布完成后，等待几十秒，打开博客地址：

```text
https://taojinlab.github.io/my-blog/
```

如果新文章出现在首页，说明发布成功。

## 常用命令总结

以后发布新文章，可以按下面这套流程操作：

```powershell
cd "D:\code data\my-blog"

npx hexo new "文章标题"
npx hexo server
```

预览确认无误后：

```powershell
git add .
git commit -m "Add new post"
git push

npx hexo clean
npx hexo generate
npx hexo deploy
```

简单记忆：

```text
新建文章 -> 编辑文章 -> 本地预览 -> 保存源码 -> 生成网页 -> 发布网站
```

## generate 和 deploy 的区别

`npx hexo generate` 只负责生成网页，不会上传 GitHub。

它的作用是：

```text
Markdown 文章 + Hexo 配置 + 主题
        ↓
public 静态网页文件
```

`npx hexo deploy` 负责发布网页。

它的作用是：

```text
public 静态网页文件
        ↓
GitHub 仓库 gh-pages 分支
        ↓
GitHub Pages 网站
```

所以，只执行 `npx hexo generate`，网站不会更新。

要让网站更新，需要执行：

```powershell
npx hexo generate
npx hexo deploy
```

如果想更稳妥，可以每次发布前都执行：

```powershell
npx hexo clean
npx hexo generate
npx hexo deploy
```
